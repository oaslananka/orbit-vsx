# Orbit README Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the existing linear README opening with a concise product landing page while preserving all user, security, and maintainer facts.

**Architecture:** Keep the change documentation-only except for a focused contract test. `README.md` owns presentation and navigation; `test/unit/readme-contract.test.ts` owns stable truth checks against `README.md` and `package.json`.

**Tech Stack:** GitHub-flavored Markdown, HTML badge markup, Node.js, TypeScript, Mocha, `node:assert`.

## Global Constraints

- Product name remains `Orbit MCP & A2A`.
- Extension identifier remains `oaslananka.orbit-vsx`.
- Current release remains `0.6.0` and must be read from `package.json` by tests.
- License remains Apache-2.0.
- Do not claim a documentation site or OpenSSF Best Practices project that does not exist.
- Companion services remain separate runtime dependencies and are not started by the extension.
- Keep CodeQL and Semgrep workflow badges tied to repository-owned workflows.
