# pnpm Override Register

`pnpm-workspace.yaml` overrides are temporary compatibility or security controls, not
ordinary dependency pins. This register is authoritative for ownership, evidence, and
removal conditions. Any pull request that adds, changes, or removes an override must
update this file and regenerate `pnpm-lock.yaml`.

## Review procedure

For an override removal pull request:

1. Run `corepack pnpm why <package>` and record every remaining parent path.
2. Remove only the candidate override and run `corepack pnpm install --lockfile-only`.
3. Run `corepack pnpm audit --audit-level moderate` and the full protected verification
   matrix.
4. Confirm the relevant Dependabot alert or compatibility condition remains resolved.
5. Update this register and the Renovate Dependency Dashboard in the same pull request.

Overrides must not be silently retained after their removal conditions become true.
